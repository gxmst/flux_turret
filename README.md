# Flux Turret - 高能光棱防御系统

![Minecraft 1.20.1](https://img.shields.io/badge/Minecraft-1.20.1-brightgreen.svg)
![Forge](https://img.shields.io/badge/Forge-47.4.10-orange.svg)
![Java 17](https://img.shields.io/badge/Java-17-blue.svg)

Flux Turret 是一个面向 Minecraft Forge 1.20.1 的 FE 驱动光棱防御模组。它的设计目标不是简单做一座自动炮塔，而是在方块实体系统里复现类似《红色警戒 2》光棱塔的核心体验：多座防御节点自动组网，选择最合适的主塔，将分布式能量汇聚成一次高强度打击。

## 核心特性

- **Forge Energy 驱动**：炮塔只接收 FE，不向外输出能量，适合接入常见科技模组的发电机、线缆和储能系统。
- **动态主塔选举**：网络会根据目标距离、炮塔能量和视线条件选择最合适的主塔，避免无电或隔墙节点错误抢占攻击权。
- **光棱中继网络**：从塔会加入主塔的支援树，逐跳中继能量并提高最终伤害，形成可扩展的阵列防御。
- **服务端权威逻辑**：目标选择、能量扣除、支援计数和伤害结算在服务端完成，客户端只负责同步后的视觉表现。
- **GeckoLib 模型渲染**：使用 GeckoLib 加载 3D 方块模型，并通过自定义渲染器绘制具有体积感的动态能量光束。
- **CI 远端验证**：仓库包含 GitHub Actions 构建流程，使用 Java 17 和 Gradle wrapper 自动编译并上传模组 jar。

## 系统设计

Flux Turret 的连锁机制可以理解为一个轻量级的方块实体网络拓扑求解问题。每座炮塔都是一个节点，节点会周期性维护邻居缓存和目标缓存；当可攻击目标出现时，满足条件的节点参与主塔选举，胜出的主塔成为 depth 0 的根节点。

从塔不会盲目提供加成。它必须具备足够能量、能接入一个有效父节点，并归属于当前主塔的支援树，才会被计入支持数量。主塔随后按父子距离逐跳遍历支援树，使用稳定的 depth 限制控制网络规模，避免不同攻击树之间串台。

这种设计把原本看似简单的“附近塔加伤害”拆成几个明确阶段：

1. **目标过滤**：扫描敌对生物，并通过视线检测剔除不可攻击目标。
2. **主塔选举**：只比较有足够主炮能量且对目标有视线的候选炮塔。
3. **中继接入**：从塔寻找已激活的低 depth 父节点，记录 masterPos 和目标父节点。
4. **支援统计**：主塔只统计同一棵树内、有电、逐跳可达的 relay。
5. **同步渲染**：服务端同步 target、lastFireTime 和支持数，客户端保留短暂光束残影。

## 机制参数

| 参数 | 数值 | 说明 |
| --- | ---: | --- |
| 能量容量 | 100,000 FE | 单座炮塔内部储能 |
| 最大输入 | 1,000 FE/t | 外部 FE 输入上限 |
| 主塔开火消耗 | 1,000 FE | 造成伤害时扣除 |
| 从塔中继消耗 | 500 FE | relay 参与中继时扣除 |
| 目标扫描范围 | 16.5 格 | 需要满足视线检测 |
| 邻居连接范围 | 12 格 | 每一跳的父子距离 |
| 最大支援深度 | 6 层 | 从主塔向外逐跳扩展 |
| 主塔冷却 | 20 tick | 约 1 秒 |
| 从塔中继冷却 | 2 tick | 高频视觉/中继反馈 |
| 伤害公式 | `10 * (1 + 支持数 * 0.5)` | 支持树越大，主炮越强 |

## 使用方式

1. 放置 `Prism Tower` 方块。
2. 使用任意兼容 FE 的线缆、管道或发电设备为炮塔充能。
3. 确保目标与主塔之间有可通过的视线。
4. 将多座炮塔按 10-12 格左右间距布置，可形成稳定的逐跳支援阵列。

## 工程要点

- **能力生命周期**：FE capability 使用可恢复的 `LazyOptional`，避免区块卸载后充能接口永久失效。
- **内部扣能模型**：外部 capability 只允许输入，内部逻辑通过专用消耗方法扣除能量，保证“只进不出”和“可以开火消耗”同时成立。
- **缓存节流**：邻居缓存和目标缓存分频刷新，降低大规模阵列每 tick 暴力扫描带来的 TPS 压力。
- **NBT 同步安全**：可空字段在加载前先清理，避免 targetPos、masterPos 等旧状态残留到客户端。
- **视觉状态清理**：当 targetType 归零或停止开火时，客户端会清空光束目标和缓存位置，减少幽灵光束。

## 开发环境

- Minecraft: 1.20.1
- Forge: 47.4.10
- Java: 17
- GeckoLib: 4.4.4
- 构建系统: Gradle + ForgeGradle 6.x

```bash
chmod +x gradlew
./gradlew build
```

GitHub Actions 会在 push、pull request 和手动触发时运行 `./gradlew --no-daemon build`，并上传 `build/libs/*.jar` 作为构建产物。

## 建议测试场景

- 单塔充能后对有视线怪物开火。
- 两座塔分别处于隔墙/无遮挡位置，确认无遮挡塔获得主塔权。
- 6 层长链路逐跳亮起，并正确提升主塔伤害。
- 从塔能量低于 500 FE 时退出支援树，恢复供电后重新接入。
- 两组阵列同时攻击不同目标时，支援树不会互相计入。
